## 4. Development Process

### How I built it:

**Step 1 — Problem Statement Selection**
I chose the Street Performer domain myself. The mapping from school equipment lending to street performance booking was my own creative decision — replacing equipment with performance slots, students with artists, and lab assistants with city authority.

**Step 2 — Architecture Design**
I decided on microservice architecture with 3 services after studying the assignment requirements. The port allocation (5001 for auth, 5002 for booking, 8000 for gateway) and database-per-service pattern were my own design decisions.

**Step 3 — Backend Development**
I wrote the Mongoose schemas field by field — deciding which fields were required, what enums to use, and which indexes to add for performance. The JWT middleware, bcrypt integration, and role-based access control were all written manually.

**Step 4 — Frontend Development**
I built the React frontend with Context API for auth state, React Router for navigation, and Axios for API calls. The dark theme CSS design system was written from scratch.

**Step 5 — Debugging**
I independently debugged multiple issues:
- `next is not a function` in Mongoose pre-save hook (async hook syntax change in Mongoose 7+)
- Express 5 wildcard syntax change (`*` → `/{*path}`)
- CORS configuration for multiple localhost ports
- MongoDB `_id` vs `id` field naming in frontend

---

## 5. Issues Encountered & How I Fixed Them

| Issue | Root Cause | My Fix |
|---|---|---|
| `next is not a function` in User.js | Mongoose 7+ async hooks don't use next() | Removed next parameter, used async/await return |
| `PathError: Missing parameter name` in gateway | Express 5 changed wildcard syntax | Changed `'*'` to `'/{*path}'` |
| CORS error from localhost:3001 | React started on 3001 when 3000 was busy | Changed CORS origin to allow any localhost port dynamically |
| `Cast to ObjectId failed` in booking | MongoDB returns `_id` not `id` | Fixed all frontend references from `.id` to `._id` |
| `Cannot POST /login` | API Gateway pathRewrite not working | Rewrote gateway using manual axios proxy instead of http-proxy-middleware |
| `ENOENT: data/db.json` | data/ directory not created | Added `fs.mkdirSync()` before any file write operations |

## 7. Reflection Summary

This project was built manually from scratch, using AI only as a reference tool — similar to how one would use official documentation or Stack Overflow. The actual architecture design, code writing, and debugging were all done independently.

The biggest learning came from debugging real errors in a microservice environment. Issues like Express 5 breaking changes, Mongoose async hook syntax, and CORS across multiple ports were all solved by reading error messages carefully and understanding the underlying cause — not by asking AI to fix them.

Building a microservice system manually gave me deep understanding of:
- How JWT tokens flow across service boundaries
- Why database-per-service is important for independent scaling
- How an API gateway acts as a single entry point and why that simplifies frontend code
- The operational complexity of running multiple services simultaneously

For future projects, I would use the same approach — AI for conceptual learning, manual coding for implementation — as it builds stronger engineering intuition than copy-pasting AI-generated code.
